SNAPSMACK Alpha v0.7.4b "Invalid Ring"
Released March 15, 2026


ANONYMOUS ENGAGEMENT

  Anonymous likes: visitors can like posts without creating a
    community account. Tracked by SHA-256 hashed IP -- no PII stored.
  Anonymous reactions: same IP-hash pattern as likes. Visitors
    can react to posts without login. Auth gate on reaction trigger
    button removed entirely.
  Guest reaction state: dock and inline component both display
    the visitor's existing reaction on page load (IP hash lookup
    with try-catch fallback for pre-migration installs).
  Max active reactions raised from 6 to 10.


TRUE GRIT

  Transparency system: header and footer backgrounds rendered
    via ::before pseudo-elements with configurable opacity (0-100
    slider) so text stays fully opaque.
  Header nav colour and hover colour options added to manifest.
  Footer font colour and link hover colour options added to manifest.
  Skin bumped to v1.1.
  optical_lift default changed from 50 to 0 (user typically sets
    this to zero).
  Static page top padding balanced with bottom (50px each).
  Footer padding stripped of !important so footer height slider
    works from manifest.


ADMIN

  Box sub-structure classes (.box-header, .box-body, .box-title)
    added to admin-theme-geometry-master.css and themed in the
    midnight-lime colour file.
  Forum page inline styles (~400 lines) extracted to geometry
    master (layout) and theme colour file (colours). No more
    inline <style> block in smack-forum.php.
  Help manual inline styles (~170 lines) extracted the same way.
    CSS variable references (var(--accent), var(--text-secondary),
    etc.) replaced with direct class selectors matching the admin
    theme system.
  Help system updates for Photogram, True Grit, and main help file.
  Category description field wired up in admin UI.


HOUSEKEEPING

  Cookie consent banner: core/consent-banner.php -- links to
    privacy/cookie page if one exists.
  Downloads simplified to global-only -- per-post allow_download
    gate removed from download.php and core/download-overlay.php.
  Canonical schema reference file tracked in database/schema/ --
    unignored from .gitignore, rebuilt with all 24 tables and
    migration columns folded in.
  Three separate migrate-074b-*.sql files consolidated into
    single migrate-074b.sql.
  Old design reference documents removed from repo.


BUG FIXES

  Community forum fatal crash on PHP 8+: count($cats) called
    before $cats was defined (line 702 vs 710). Page rendered
    the header then died silently. Fixed by moving the assignment
    above the count call.
  Help manual search input focus border was blue (#6cf from
    orphaned CSS variable fallback) instead of neon green (#39FF14).
    Now themed through the midnight-lime colour file.
  Consent banner fatal PDOException: query referenced non-existent
    columns page_slug and page_status -- corrected to slug and
    is_active.
  Reaction trigger button (smiley face) forced login redirect
    even though likes (heart) worked anonymously. Root cause: JS
    auth gate on #ss-cdock-react-trigger not removed when anonymous
    likes were added. Fixed by removing all auth redirects from
    reaction triggers in ss-engine-community.js.


SCHEMA CHANGES

  migrate-074b.sql: adds edited_at to snap_community_comments,
    adds guest_hash column + index to both snap_likes and
    snap_reactions. Fully idempotent -- checks column/index
    existence via INFORMATION_SCHEMA before altering. Safe to
    re-run on any install state.
